# Cantonese ASR Fine-Tuning

Fine-tune wav2vec2 and Whisper models on Common Voice Cantonese (yue) for automatic speech recognition.

Running base `ctl/wav2vec2-large-xlsr-cantonese` on `cv-corpus-25.0-2026-03-09 zh-HK` has CER: 14.40.

This repo attempts to recreate and then refine the fine-tuned Cantonese models that can be used with WhisperX. Method and code is based off of [khleeloo/whisper-large-v3-cantonese](https://huggingface.co/khleeloo/whisper-large-v3-cantonese) and [ctl/wav2vec2-large-xlsr-cantonese](https://huggingface.co/ctl/wav2vec2-large-xlsr-cantonese). Code is mostly generated by claude.ai but has been manually reviewed.

## Setup

```bash
pip install -r requirements.txt
```

Requires Python 3.10+ and a CUDA GPU for training (bf16 is used when available).

## Training

### Whisper

```bash
python3 train_whisper.py \
    --dataset_path data/cv-corpus-25.0/yue \
    --model openai/whisper-large-v3-turbo \
    --lr 1e-5 --epochs 10 --warmup 500 \
    --train_batch_size 4 --grad_accum 4
```

Key options:
- `--freeze_encoder` / `--unfreeze_encoder_layers N` -- control which encoder layers are trainable
- `--encoder_lr 5e-6` -- separate learning rate for encoder vs decoder
- `--lora --lora_r 16 --lora_alpha 32` -- LoRA fine-tuning (requires `peft`)
- `--lr_schedule tri_stage` -- fairseq-style warmup/hold/decay schedule (see below)
- `--nopunct_in_eval` -- use punctuation-normalized CER for checkpoint selection
- `--early_stopping_patience 5` -- stop if CER doesn't improve for N eval rounds

### wav2vec2

```bash
python3 train_wav2vec2.py \
    --dataset_path data/cv-corpus-25.0/yue \
    --model facebook/wav2vec2-xls-r-1b \
    --lr 5e-5 --epochs 15 --warmup 1000 \
    --train_batch_size 4 --grad_accum 4
```

Key options:
- `--unfreeze` -- unfreeze the feature encoder (frozen by default)
- `--compile` -- use `torch.compile` (experimental)
- `--lr_schedule tri_stage` -- fairseq-style warmup/hold/decay schedule
- `--attention_dropout 0.1 --hidden_dropout 0.1 --layerdrop 0.1` -- regularization

### LR schedules

Both training scripts support two learning rate schedules via `--lr_schedule`:

- **cosine** (default): linear warmup then cosine decay to near-zero
- **tri_stage**: fairseq-style linear warmup, constant hold at peak LR, then linear decay

Tri-stage options (ignored when using cosine):
```
--tri_stage_warmup_pct 0.1   # fraction of steps for warmup
--tri_stage_hold_pct 0.4     # fraction of steps at peak LR
--tri_stage_final_lr_scale 0.05  # final LR as fraction of peak
```

### Dataset options (shared)

Both training scripts use `create_splits.py` to build deterministic train/validation/test splits from Common Voice TSV files.

- `--dataset_path path1,path2` -- comma-separated paths to dataset directories (each with `clips/` and TSVs)
- `--all_tsv validated.tsv` -- TSV file(s) containing all usable samples
- `--holdback_tsv test.tsv` -- exclude these samples from training (for fair evaluation)
- `--pct_validation 0.1 --pct_test 0.1` -- split fractions
- `--write_splits` -- write split TSV files to the current directory
- `--no_streaming` -- preprocess all data upfront (uses disk cache, faster training)
- `--max_train_samples N` / `--max_eval_samples N` -- subsample for quick experiments

## Evaluation

Evaluate a trained model on test and holdback splits:

```bash
# Whisper
python3 eval_whisper.py --model_path whisper-finetuned/final \
    --dataset_path data/cv-corpus-25.0/yue

# wav2vec2
python3 eval_wav2vec2.py --model_path wav2vec2-xls-r-1b-cantonese-yue/final \
    --dataset_path data/cv-corpus-25.0/yue
```

## Utilities

| File | Purpose |
|---|---|
| `cer_utils.py` | Shared CER computation with jiwer text normalization |
| `create_splits.py` | Deterministic train/val/test splits from Common Voice TSVs |
| `lr_schedule.py` | Tri-stage LR schedule (warmup/hold/decay) shared by both trainers |
| `build_vocab.py` | Build character vocabulary (`vocab.json`) for wav2vec2 |
| `build_processor.py` | Create and save a `Wav2Vec2Processor` from vocab + feature extractor |
| `upload_model.py` | Upload trained models to HuggingFace Hub with auto-generated model cards |
| `eval.py` | Quick wav2vec2 evaluation script |
